技術與設計說明

✨ 設計重點5 項
  • 🔌

    MCP 標準化讓 AI 直接取數

    把 GA4 / GSC / GTM 的查詢包成標準化工具介面，AI 不必處理各家 API 細節，能直接「要什麼指標就取什麼指標」。

  • 🔗

    跨來源資料整合

    流量（GA4）、自然搜尋（GSC）與事件設定（GTM）合在同一份脈絡裡解讀，洞察比單看任一來源更完整。

  • 🌡️

    紅漲綠跌溫度圖

    沿用台股慣例：紅＝成長／正向、綠＝下滑／負向，表格依數值深淺上色，主管一眼看出強弱與變化。

  • 🧪

    脫敏資料產生器

    另寫產生器把真實快照轉成結構相同、數字虛構的展示資料，作品集可公開又不洩任何真實數據。

  • 🌐

    中英雙語、同一套邏輯

    儀表板中英版共用同一套生成邏輯，避免兩份各自維護造成數字或結構不一致。

🛠️ 在這個專案我實際做了什麼6 項 · 點開

這是一個從串接 Google 數據服務、自建 MCP、到 AI 自動產出可視化分析都自行完成的數據分析專案， 重點在「讓 AI 能直接、可靠地取用多來源行銷數據並產出洞察」。

  • 🔌自建 MCP Server 串 GA4 / GSC / GTM

    把 GA4 Data API、Search Console API 與 Tag Manager 的取數封裝成 MCP 標準化工具，讓 AI 以一致介面取得跨來源數據，不必各自處理授權與回應格式。

  • 🧪設計脫敏資料產生器

    為了公開作品集，另寫產生器把真實快照轉為結構相同、數值虛構的假數據， 真實版與展示版共用同一套生成邏輯，確保版面一致又不外洩任何真實數據。

  • 🤖AI 自動產出分析

    由 AI 讀取整合後的指標，產出成效摘要、SEO 機會與轉換洞察等可讀報告， 把「數字」轉成「人看得懂、可行動的建議」。

  • 📈資料視覺化儀表板（中英）

    建置 KPI 卡、進站頁排名、走勢與轉換漏斗等視覺化模組， 並提供中英雙語版本，方便對內外不同對象呈現。

  • 🌡️紅綠慣例與溫度圖

    採台股紅漲綠跌慣例與溫度圖上色，讓成長／下滑、強弱在表格中一眼可辨， 降低閱讀門檻、加速決策。

  • 🏗️自動化重生與發布

    把假數據重生、版面建置與發布串成可重複執行的流程， 更新只需一鍵重跑，不必手改成品 HTML，維護成本低且不易出錯。
