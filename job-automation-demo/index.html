技術與設計說明

✨ 設計重點5 項
  • 🕷️

    多平台爬蟲 + 反爬處理

    同時爬多個求職平台，處理各家參數差異與防爬機制（如先打首頁取 cookie），輸出統一格式。

  • ⚡

    規則評分秒級、AI 評分加值

    先用純規則秒級評分（無需 API），再用 AI 對重點職缺加值分析，兼顧速度與品質。

  • 📄

    JD 自動抓取與解析

    自動補抓職缺詳細 JD，作為後續 AI 生成 CV / QA 的依據，減少人工複製貼上。

  • 🤖

    AI 客製生成

    依每個職缺的 JD 生成對應的客製 CV 重點與面試 QA，讓每份應徵都對症下藥。

  • 🗂️

    Notion 去重同步

    把職缺與進度寫入 Notion 追蹤，並去重避免重複投遞，求職進度一目了然。

🛠️ 在這個專案我實際做了什麼6 項 · 點開

從 多來源爬蟲、評分、AI 生成到 Notion 自動化都自行完成的求職自動化 pipeline。

  • 🕷️多來源爬蟲 pipeline

    設計 104 / 1111 等多平台爬蟲，處理 Cloudflare cookie、參數差異與分頁，輸出統一 JSON 並可擴充新平台。

  • ⭐規則 + AI 雙層評分

    先以純規則合併多平台輸出秒級評分（無需 API），再用 AI 對高分職缺加值，分成 A / B / C。

  • 📄JD 解析

    自動補抓各平台職缺 JD（部分平台 description 已內含、部分需另抓），作為生成依據。

  • 🤖AI 生成 CV / 面試 QA

    依 JD 用 LLM 產出客製化履歷重點與面試問答，提升投遞效率與命中率。

  • 🗂️Notion API 自動化

    以 Notion API 把職缺與進度寫入資料庫並去重，建立可追蹤的求職看板。

  • 🏗️可擴充 pipeline 架構

    主流程一鍵跑全平台（`--source all`），模組化設計方便加入新爬蟲與新步驟。
