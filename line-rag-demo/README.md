# LINE 智能客服機器人_展示用 (MVP)

串接 **LINE + n8n + RAG 向量資料庫 + AI** 的 B2B 禮品客服機器人。本頁為**靜態前端展示**，
純粹呈現介面、互動與系統架構流程，**資料為虛構範例、AI 回應為前端模擬**，不串接任何真實服務。

> 📌 公開作品展示頁，僅含前端與少量範例資料；不含任何金鑰、真實資料或營運邏輯。

## 線上展示

直接以瀏覽器開啟 `index.html`；若資源載入異常，改用本機伺服器：

```bash
python -m http.server 8000   # 開 http://localhost:8000
```

## 功能特色

- **架構流程動畫**：送出訊息後，LINE → n8n → 路由判斷 → RAG 檢索 → AI 生成 → 回覆/通知 依序亮燈示意
- **模擬聊天**：產品編號精準查詢、模糊推薦、FAQ、無牌價保守處理、混合問題等情境
- **冷熱資料分層**與**分層檢索**（先精準、找不到才語意搜尋）的概念呈現
- **處理流程耗時面板**（模擬），呈現各環節時間佔比
- 深色專業介面、響應式版面

## 在這個專案我實際做了什麼（高層說明）

從**系統設計、外部服務整合到生產化維運**皆自行完成的 B2B 客服機器人：

- 🔗 **多平台整合**：串接 LINE Messaging API、n8n、向量資料庫與 AI 生成模型，整合宜搭表單與內部群組通知，串起「客戶詢問 → 自動回覆 → 內部跟進」完整閉環。
- 🧠 **RAG 檢索式生成**：將 FAQ／產品知識向量化建索引，依語意檢索後再生成回答，讓回覆有所本、可控、可追溯。
- ⚡ **檢索策略與成本最佳化**：多層檢索（精準優先、語意後備）＋ 冷熱資料分層與本地快取，同時改善回應速度與 API 成本。
- 🛡️ **商業風險控管**：報價採保守策略，無對應牌價即轉人工並自動建單，把「AI 答錯的營運風險」納入設計。
- 🏗️ **生產化與工程紀律**：處理 webhook 簽章驗證、訊息去抖動、增量同步與本地快照索引，並以自動化測試守住關鍵行為確保穩定。
- 🔁 **從低程式碼到工程實作**：先用 n8n 快速驗證上線，再為效能與可控性重構為可維護服務。

> 實際的檢索邏輯、資料結構、prompt 與門檻參數屬營運 know-how，未公開於此展示，可於面談時進一步說明。
