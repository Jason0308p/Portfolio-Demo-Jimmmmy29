技術與設計說明

✨ 設計重點5 項
  • 🔌

    整合多個宜搭 API 成自建 MCP

    把宜搭多支表單 API（查詢／新增／更新 J1、J1A）整合封裝成自建 MCP 工具，讓 AI 以一致介面讀待處理商品並回寫結果，不必人工搬資料。

  • 🛡️

    AI 生成 grounding / 防幻覺

    生成內容嚴格 grounding 在原始商品資料與圖片，並用規則過濾「最好／第一」等誇大或杜撰規格的字詞，降低 AI 幻覺、確保內容可追溯。

  • 🔍

    SEO + GEO 文案策略

    名稱核心關鍵字前置、敘述用列點＋自然語言句式，兼顧搜尋引擎與生成式搜尋（GEO）。

  • ⚙️

    批次 diff + preflight

    先預檢擋掉缺資料/錯欄位，只更新真正有變動的欄位，API 失敗有限次重試。

  • 🔁

    n8n 串接社群與回收

    處理完成自動發社群貼文，再把觸及/互動等成效回收寫回追蹤，形成閉環。

🛠️ 在這個專案我實際做了什麼6 項

從 自建宜搭 MCP、AI 內容生成、批次同步到 n8n 社群自動化都自行完成的電商上架自動化專案。

  • 🔌整合多個宜搭 API 自建 MCP Server

    將宜搭 (Yida) 多支表單 API 整合封裝成自建 MCP server，讓 AI 以一致介面讀取 J1 新品表單、回寫 J1 / J1A，支援 insertOrUpdate 與最新版本更新。

  • 🤖AI SEO / GEO 內容生成

    依爬蟲商品原始資料生成 SEO 商品名、純文字敘述、HTML 敘述、標籤，採關鍵字前置與 GEO 句式。

  • 🖼️自動補圖 / 補敘述

    針對缺圖缺敘述的爬蟲商品，自動補上商品圖與結構化敘述，讓陽春資料變成可上架內容。

  • ⚙️批次工具：diff / preflight / 重試

    把一次性腳本整理成可配置、可預檢、只更新差異、可重試的批次同步流程，並落地中繼 JSON / 報表 / 統計。

  • 📣n8n 社群自動化與成效回收

    用 n8n 把上架完成的商品自動發到社群，並把觸及、互動、點擊等成效回收寫回追蹤表。

  • 🛡️AI 生成防幻覺（grounding）與可追溯

    生成內容嚴格 grounding 在原始商品資料，規則過濾誇大／杜撰規格；每筆記錄使用模型與結果供稽核追溯。
