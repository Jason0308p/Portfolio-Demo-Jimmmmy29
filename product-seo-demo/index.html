  "@context": "https://schema.org",
  "@type": "Product",
  "name": "Sample Product",
  "sku": "product-abc",
  "url": "https://example-shop.com/products/product-abc",
  "offers": {
    "@type": "Offer",
    "priceCurrency": "TWD",
    "price": "999",
    "availability": "https://schema.org/InStock"
  }
}

GEO（生成式搜尋優化）策略

  • 內容結構化為可引用事實：條列規格、明確數字，方便 AI 摘要直接引用。
  • FAQ 內容搭配 FAQPage schema，提高被生成式問答引用的機率。
  • 品牌實體資訊（名稱／網域／社群）保持一致，強化 AI 對品牌的辨識。
  • 伺服器渲染 Liquid 模板，確保 AI 爬蟲可直接讀到完整內容，不依賴前端渲染。

技術與設計說明

✨ 設計重點5 項
  • 🧩

    用官方工具不重造輪子

    商品／頁面／部落格操作走 Shopify CLI＋官方 dev-mcp；宜搭端維持自建 MCP，兩邊各司其職，不把官方已提供的能力重新造一遍。

  • ⚙️

    批次效能設計

    用 ThreadPoolExecutor 平行處理搭配批次 GraphQL 查詢（handle:a OR handle:b），取代逐筆序列請求，大幅壓縮批次作業時間。

  • 🛡️

    安全發布：patch 不覆寫

    頁面更新流程為備份 → 切片 patch → 驗證 → 寫入 → 重新讀取驗證，避免整頁覆寫造成內容遺失或版型跑掉。

  • 🔍

    SEO＋GEO 雙軌內容策略

    JSON-LD 覆蓋主要型別（Product／Article／FAQPage 等），內容同時寫成可引用事實列點，兼顧傳統搜尋與生成式搜尋。

  • 🔁

    資料一致性與稽核

    寫入前重讀最新資料避免覆蓋他人變更，寫入後重讀驗證結果，留下可追溯紀錄。

🛠️ 在這個專案我實際做了什麼6 項

獨立完成的 Shopify 商品自動化：以 Shopify CLI＋Admin GraphQL 驅動商品／集合／頁面／部落格批次管理，官方 dev-mcp 輔助開發查詢，並把宜搭表單接為下架流程的下游同步，加上搜尋意圖分類與加權再排序提升商品可被找到的機率。

  • 🧰Shopify CLI＋Admin GraphQL 批次工具

    用 Shopify CLI 執行 Admin GraphQL 查詢／異動（--query-file／--variable-file／--allow-mutations），並串官方 dev-mcp 輔助查詢 schema 與文件，加速開發除錯。

  • ⚡批次下架效能優化＋宜搭下游同步

    Excel 名單觸發批次 productUpdate(status:ARCHIVED)，序列處理改成 ThreadPoolExecutor 平行處理＋批次 handle 查詢，示意效能約 15 分鐘 → 1.6 分鐘，下架同時同步宜搭表單狀態。

  • 🖼️Collection 縮圖 AI 生成 Pipeline

    Prompt 生成 → AI 繪圖 → stagedUploadsCreate → collectionUpdate → 重新讀取驗證上線，全流程自動化不需人工上傳。

  • 🧱頁面產生器（兩套版型＋區塊庫＋安全發布）

    購買指南型／產業著陸頁型兩套版型，共用 Hero／TOC／FAQ／CTA／比較表／Sticky Bar 區塊庫，發布走備份→切片 patch→驗證→寫入→重新讀取驗證，不整頁覆寫。

  • 📝部落格文章與 SEO/GEO 內容規則

    生成文章標題與 70–80 全形字 meta description，規則過濾不捏造規格數字、避免句式公式化重複。

  • 🎯搜尋與推薦：意圖分類＋加權再排序

    關鍵字比對意圖分類（保溫瓶／馬克杯／帆布提袋等約 8 類）；搜尋結果依標題 +20／供應商或類型 +10／Meta +3 加權重排序，並正規化半形／全形空白差異。
